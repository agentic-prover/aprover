narrows confidence to "no bugs within bound k under the encoded

## bmc-agent's outcome taxonomy

For each function under test, bmc-agent now produces one of:

1. **REAL BUG**: confirmed_dynamic (UBSan/ASan triggered) >
   confirmed_system_entry > confirmed_bmc > realistic. → triage,
   build PoC, submit.

2. **VERIFIED CLEAN (within bound k)**: no CE, OR all CEs auto-rejected
   by witness-pattern / dynamic-no-trigger / LLM realism /
   feedback-loop convergence. → record positive verification.

3. **UNCERTAIN**: CE found but realism returned UNCERTAIN, OR unwind
   exhausted, OR dynamic INCONCLUSIVE. → human review, OR loop
   iterates, OR bump unwind + re-run.

4. **TOOL FAILURE**: CBMC frontend error, LLM API failure, harness
   compile error, etc. → arm-(a) code fix to bmc-agent itself.
