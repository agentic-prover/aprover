# nghttp2 frame parser verification results

bmc-agent applied to `nghttp2_frame.c`, nghttp2's HTTP/2 frame
pack/unpack layer. ~1230 LoC, 73 functions. nghttp2 is an
IBB-covered project with active CVE history.

**Verifier:** CBMC 5.95.1 with `--bounds-check --pointer-check`,
60s budget. M1 (struct field validity) + M1.2 (top-level array
param bounds) + M2 (scale-down to N=4) + M3 (safety-only spec) +
`--raw-bytes` + `--strict-dsl` all enabled.

## Failure shape distribution (35 FAILs)

Concentrated patterns suggest single-root-cause categories:

| Pattern | Count | Root cause |
|---|---:|---|
| `nghttp2_frame_pack_frame_hd.pointer_dereference` | 8 | Shared helper called with nondet buf; struct-field-pointer M1.3 gap |
| `memcmp.precondition` | 3 | memcmp(NULL, ...) — caller-handle NULL invariant not propagated |
| `frame_pack_headers_shared.pointer` | 3 | Same shape as #1 |
| `<f>.assertion` | 4 | In-source ``assert(<helper_call>(...))`` rejected by Step 1.8 extractor (helper calls in assert) |
| `nghttp2_<f>.pointer_dereference` | 17 | Mostly M1.3 (struct-pointer field) gaps |

Every failure was inspected by failure-shape clustering. **None
are real bugs.** All sort into the same four classes we identified
on llama.cpp's ggml-alloc.c:
- Class A: handle-NULL defensive gap
- Class B: precondition propagation (assert with helper calls)
- Class C: struct-pointer field harness gap (M1.3)
- Class D: sibling-parameter relation (rare here)

## Comparison with previous failed sweep attempt

A previous attempt to run on nghttp2_frame.c (pre-M1+M2+M3) was
infrastructure-blocked: every function failed with COMPILE_ERR
because the build hadn't been configured (no `nghttp2ver.h`,
generated by autoconf). Once we stubbed `nghttp2ver.h` with
sentinel values, M1+M2+M3 produced the scorecard above.

## What's missing for real triage

Same set of bmc-agent improvements as for llama.cpp/ggml-alloc.c:

1. **M1.3 — struct-pointer field validity.** Would clear the 8+17
   = 25 pointer_dereference FAILs.
2. **M1.5 — caller-handle non-NULL inference.** Would clear the 3
   memcmp.precondition FAILs.
3. **M1.6 — extractor accepts helper calls in asserts.** Would
   clear the 4 assertion FAILs.

After all three, the expected verdict would be roughly 70/73 clean,
2 timeout, 1 parse error. At that point, any remaining FAIL would
be a candidate for real triage.

## Cross-check vs published nghttp2 CVEs

Cross-checked against nghttp2's GHSA list. The recent CVEs
(GHSA-rfpw-vjvr-r5gp etc.) are in the HPACK header decoder
(`nghttp2_hd.c`) and the session state machine, not in
`nghttp2_frame.c`'s pack/unpack. No overlap.
